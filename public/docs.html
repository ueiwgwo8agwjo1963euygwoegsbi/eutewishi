Overview & Quick Start

This is a secure, rate-limited public API for sending SMS messages using M2Tech Tronix service. The API is designed for high reliability and includes robust security features like host-based protection, rate limiting, CORS protection, and input validation.

Base Endpoint

https://jrmph-freesmsapi-bvyo.onrender.com

SMS Provider

M2Tech Tronix Integration (No External API Keys Required)

Security Features

üîí Host Protection

Only authorized domains can access the API. Prevents unauthorized hosting and cloning.

‚ö° Rate Limiting

100 requests per 15 minutes per IP address to prevent abuse.

Endpoint: Send SMS

POST /api/send-sms

Initiates the transmission of an SMS message to a specified mobile number using M2Tech service. All inputs are validated for format and length.

Request Body (application/json)

{
    "to": "+639123456789",
    "message": "Hello from JRM FreeSMS API!",
    "senderName": "MyApp"
}

Parameters

Parameter Type Req. Description
to string YES Target phone number. Supports international format (+63) or local (09xxxxxxxxx).
message string YES The message content (max 160 characters).
senderName string NO Sender name/identifier for the message (optional, defaults to 'FreeSMS').

Success Response (200 OK)

{
    "success": true,
    "message": "SMS sent successfully",
    "timestamp": "2025-12-11T21:22:36.000Z",
    "to": "+639123456789",
    "responseTime": "245ms",
    "metadata": {
        "messageLength": 32,
        "senderName": "MyApp"
    }
}

Error Responses

403 Forbidden (Access Denied)

{
    "error": "Access Denied",
    "message": "This API is only available from authorized domains.",
    "yourHost": "unauthorized-domain.com",
    "allowedHosts": ["jrmph-freesmsapi-bvyo.onrender.com"]
}

429 Too Many Requests

{
    "error": "Too Many Requests",
    "message": "Rate limit exceeded. Try again later.",
    "retryAfter": 900
}

400 Bad Request

{
    "success": false,
    "error": "Sender name required",
    "message": "Sender name cannot be empty"
}

Code Examples

Easily integrate the POST /api/send-sms endpoint into your application using common technologies.

-H "Content-Type: application/json" \
-d '{
    "to": "+639123456789",
    "message": "Hello from JRM FreeSMS API!",
    "senderName": "MyApp"
}'

JavaScript (Fetch)

const response = await fetch('https://jrmph-freesmsapi-bvyo.onrender.com/api/send-sms', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        to: '+639123456789',
        message: 'Hello from JRM FreeSMS API!',
        senderName: 'MyApp'
    })
});

const result = await response.json();
console.log(result);

Python (requests)

import requests

url = "https://jrmph-freesmsapi-bvyo.onrender.com/api/send-sms"
data = {
    "to": "+639123456789",
    "message": "Hello from JRM FreeSMS API!",
    "senderName": "MyApp"
}

response = requests.post(url, json=data)
print(response.json())

PHP (cURL)

$url = "https://jrmph-freesmsapi-bvyo.onrender.com/api/send-sms";
$data = [
    "to" => "+639123456789",
    "message" => "Hello from JRM FreeSMS API!",
    "senderName" => "MyApp"
];

$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, $url);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, ['Content-Type: application/json']);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
curl_close($ch);

echo $response;
